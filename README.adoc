= AsciiDoc Template

AsciiDocドキュメントをHTMLおよびPDFに変換するためのテンプレートリポジトリです。

== 概要

このリポジトリは、AsciiDocを使用したドキュメント作成のためのテンプレートを提供します。
Dockerコンテナを使用してビルド環境を統一し、GitHub Actionsで自動ビルドを実行できます。

== 機能

* AsciiDocからHTMLへの変換
* AsciiDocからPDFへの変換（日本語対応）
* Mermaid図のサポート
* GitHub Actionsによる自動ビルド

== ディレクトリ構成

[source]
----
.
├── .github/
│   └── workflows/
│       └── build-html-pdf.yml  # GitHub Actions ワークフロー
├── docs/
│   ├── ja/                     # 日本語ドキュメント
│   │   ├── config.adoc         # AsciiDoc設定（日本語）
│   │   └── index.adoc          # メインドキュメント（日本語）
│   ├── en/                     # 英語ドキュメント
│   │   ├── config.adoc         # AsciiDoc設定（英語）
│   │   └── index.adoc          # メインドキュメント（英語）
│   ├── images/                 # 共有画像
│   ├── pdf-configs/
│   │   ├── config.js           # PDF設定
│   │   └── style.yml           # PDFスタイル設定
│   └── puppeteer-config.json   # Puppeteer設定（Mermaid用）
├── scripts/
│   └── build.sh                # ビルドスクリプト（多言語対応）
├── Dockerfile                  # ビルド用Dockerイメージ
└── README.adoc                 # このファイル
----

== 使い方

=== ローカルでのビルド

Dockerを使用してビルドします:

[source,bash]
----
# Dockerイメージをビルド
docker build -t adoc-builder .

# ドキュメントをビルド
docker run --rm -v $(pwd)/docs:/docs:ro -v $(pwd)/output:/output adoc-builder
----

ビルド成果物は `output/ja/` と `output/en/` ディレクトリに出力されます。

=== GitHub Actionsでのビルド

以下のいずれかの方法でビルドを実行できます:

. **タグプッシュ時**: `v*.*.*` 形式のタグをプッシュすると自動的にビルドが実行されます
. **手動実行**: GitHub Actionsの「Run workflow」ボタンから手動で実行できます

ビルド成果物はArtifactsとしてダウンロードできます。

== カスタマイズ

=== ドキュメントの編集

`docs/ja/index.adoc`（日本語）または `docs/en/index.adoc`（英語）を編集してドキュメントの内容を変更します。
共通設定は各言語ディレクトリの `config.adoc` で管理されています。

=== PDFスタイルの変更

`docs/pdf-configs/style.yml` を編集してPDFの見た目をカスタマイズできます。

= AsciiDoc Template (English)

A template repository for converting AsciiDoc documents to HTML and PDF.

== Overview

This repository provides a template for creating documentation using AsciiDoc.
It uses Docker containers to unify the build environment and enables automated builds with GitHub Actions.

== Features

* Convert AsciiDoc to HTML
* Convert AsciiDoc to PDF (with Japanese support)
* Mermaid diagram support
* Automated builds with GitHub Actions

== Directory Structure

[source]
----
.
├── .github/
│   └── workflows/
│       └── build-html-pdf.yml  # GitHub Actions workflow
├── docs/
│   ├── ja/                     # Japanese documents
│   │   ├── config.adoc         # AsciiDoc configuration (Japanese)
│   │   └── index.adoc          # Main document (Japanese)
│   ├── en/                     # English documents
│   │   ├── config.adoc         # AsciiDoc configuration (English)
│   │   └── index.adoc          # Main document (English)
│   ├── images/                 # Shared images
│   ├── pdf-configs/
│   │   ├── config.js           # PDF configuration
│   │   └── style.yml           # PDF style configuration
│   └── puppeteer-config.json   # Puppeteer configuration (for Mermaid)
├── scripts/
│   └── build.sh                # Build script (multilingual support)
├── Dockerfile                  # Docker image for building
└── README.adoc                 # This file
----

== Usage

=== Local Build

Build using Docker:

[source,bash]
----
# Build Docker image
docker build -t adoc-builder .

# Build documents
docker run --rm -v $(pwd)/docs:/docs:ro -v $(pwd)/output:/output adoc-builder
----

Build artifacts are output to `output/ja/` and `output/en/` directories.

=== Build with GitHub Actions

You can run the build in one of the following ways:

. **On tag push**: Pushing a tag in `v*.*.*` format automatically triggers the build
. **Manual execution**: You can manually run it from the "Run workflow" button in GitHub Actions

Build artifacts can be downloaded as Artifacts.

== Customization

=== Editing Documents

Edit `docs/ja/index.adoc` (Japanese) or `docs/en/index.adoc` (English) to change the document content.
Common settings are managed in `config.adoc` in each language directory.

=== Changing PDF Style

Edit `docs/pdf-configs/style.yml` to customize the PDF appearance.
